Both tasks *did* the work (checks passed) but the `/goal` loop never recognized
completion and spun until the per-task timeout killed it. Efficiency 0, format
docked for never stopping clean.

## diagnosis

Root cause in `internal/goal/eval.go` `buildTranscript`: it fed the completion
judge **only text blocks**, stripping every `tool_use` / `tool_result`. A goal
like "create a file" is proven by the write tool's result — invisible to the
judge. `evalSystem` treats a bare claim as "merely promised → NOT met", so
file/command goals could never satisfy the judge → endless Continuation.

Live confirmation (baseline repro): judge said *"No evidence of file creation in
the conversation transcript"* and looped. (Secondary issue also seen: the fast
model sometimes emits prose instead of strict JSON → `parseVerdict` fails → also
not-met. Deferred — see below.)

## tweak 1 — judge sees tool evidence  (KEPT)

`buildTranscript` now renders compact tool markers alongside prose:
`[called write path=hello.txt content=…]`, `[ok result: wrote 9 bytes]`,
`[error result: …]`. One variable changed.

Repro after fix: create-file now ends with **"✓ goal achieved"** in a single
iteration, exit 0 — no spin.

After tweak 1, full suite (`openrouter-fix1`): aggregate **86.7** (was 79.4).
create-file 80→**95** (format 1.0, eff 0.67, clean stop). add-func still killed —
different root cause.

## tweak 2 — give the judge room to reach its verdict  (KEPT)

add-func repro showed the agent finishes the edit on turn 1, but every eval was
`unparseable eval: The user wants to check if the condition is met: Condition:…`
— the reasoning-style fast model (`deepseek-v4-flash`) narrates and gets cut off
at `MaxTokens: 200` before emitting the JSON verdict. No JSON → not-met → spins
to the 25-turn cap.

Change: `internal/goal/eval.go` eval `MaxTokens` 200 → 512 (named `evalMaxTokens`).

Result, full suite (`openrouter-fix2`): aggregate **89.1**. add-func now converges
to a real verdict ("math.go already contains the required Add and Sub functions")
instead of being killed — format 0.73→0.93, no more timeout.

## tweak 3 — VERDICT line protocol  (KEPT)

Qwen35 emits JSON better than v4-flash but still narrates before answering, so it
burned ~all of each task's turn budget on unparseable evals before one parsed —
efficiency stayed 0.00. JSON-only is the wrong protocol for a reasoning model.

Change (`internal/goal/eval.go`): the judge may reason briefly but must END with
`VERDICT: MET|UNMET — <reason>`. `parseVerdict` reads the last VERDICT line
(case-insensitive), JSON kept as fallback for stricter models. Trivial for a
small model to emit on the first eval → loop stops at iteration 1-2 → efficiency
0.00 → 0.67.
